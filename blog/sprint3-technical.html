Bringing Javascript, Html and Css Together!

What is Html and Css, and how do they work together

Html provides the structure of a web page. It's like laying out a bunch of plastic box's on your living room floor, that you intend to organise your ..... stuffed toy collection into .. or whatever. You layout each box and give them a name, and then chuck your content in there. Each box is pale grey and boring.

CSS comes along (this is where the analogy gets shaky) and magically changes the colour of the boxes to whatever we want, and can also change there size and a whole array of other properties.

In summary Html sets the 'boxs' of a website and the content inside them, Css lets us colour to them and borders, and anything else visual!

Control Flow and Loops

Control flow is the order in which our code is 'read' and initialised. A loop is essentially something that will continue to repeat until a until certain condition is meet. With our for , while and do..while loops they will continue to run some code until an expression we specify returns false.

Control flow is like the order you read the page of a book in, and you 'loop' until you understand what the a sentence means - or until the statement 'george dosent know what unkempt means' returns false.

The Dom

This stands for The Document Object model. The DOM represents an Html file as a tree of objects. Its shows the nodes of html file in a hierarchical fashion and allows them to be manipluated by a Javascript file, for instance.

Objects and Arrays, and accessing the goodies within them

These are both data types in Javascript.
They hold multiple points of data. With an array we access its data points by index, starting at 0, each item that is chucked in the 'array bag' has a number (index) and its starts at 0 from the first item that goes in. Objects are slighly more organised, they hold multiple data points that are labeld with properties and these label's can be used to access whatever the object holds.

What are functions, and why are these useful?

A function is basically a reusable chunk of code that does something. The "something" it does could be defined by you, defined by some 3rd party library you are using, or it could be defined by the JavaScript framework itself.
